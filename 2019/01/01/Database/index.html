    (1) DW vs. ER

    (2) OLTP vs. DW

    (3) Why OLTP and DW models need to be different?

    • Traditional databases are not optimized for data access.They ensure integrity of data. | Data warehouse users need only read access but, need the access to be fast over a large volume of data.so denormilized.
    • Traditional Databases generally deal with two-dimensional data (tables with rows and columns): This is good for avoiding inconsistencies with frequent updates.
      DW : querying performance in a multi-dimensional data storage model is much more efficient.The data can be directly queried in any combination of
      dimensions, bypassing complex database queries.
    • Data required for data warehouse analysis comes from multiple databases

    (4) DW definition

      1. Subject-Oriented: organized around the major subjects of the enterprise (e.g., customers, products, sales) rather than the major application areas

      2. Integrated: integrates application-oriented data (OLTP) from different source systems, which often includes data that is inconsistent (e.g. in different formats).

      3. Time-variant: historical data is kept in a data warehouse

      4. Non-volatile: data is not real-time updated but is refreshed on a regular basis. New data is always added as a supplement to the database (contains historical data)

    (3) Term Frequencies

    Term Frequencies : the number of occurrences of a term in a document. Sometimes normalized with total number of occurrences in the document.

    • Collection frequency (cf ) : total number of occurrences of a term in the collection

    • Document frequency (df ) : Number of documents in the collection that contain a term

    • tf × idf (or tf .idf ) measure combines

      • term frequency (tf ) : number of times term occurs in document

      • inverse document frequency (idf ) : measure of informativeness of a term: its rarity across the whole corpus . idf = log(N/df)

      • idf 特点：

        1 highest when t occurs many times within a small number of documents;

        2 lower when the term occurs fewer times in a document, or occurs in many documents;

        3 lowest when the term occurs in virtually all documents (idf = log(1) = 0 最小为0)

    (4) parsing a document

    Tokenization : 1⃣️dropping commen terms(stop words); 2⃣️normalize terms in indexed text and query terms into the same form; 3⃣️thesauri:handle synonyms and homonyms；4⃣️ lemmatization & stemming : grammatical ,use different forms of a word. lemmatization doing proper reduction to dictionary headword form stemming reduce terms to their “roots” before indexing(affix)

    6 NoSQL(2)

    (2) NoSQL definition

    • Not only rows in tables, different formats(son,xml) —NoSQL systems store and retrieve data from many formats

    • Join-free —NoSQL systems allow you to extract your data using simple interfaces without joins

    • Schema-free —NoSQL systems allow you to drag-and-drop your data into a folder and then query it without creating an entity-relational model.

    • Many processors —NoSQL systems allow you to store your database on multiple processors and maintain high-speed performance

    • NoSQL systems create simple applications that distribute the required features across the network (APIs)
